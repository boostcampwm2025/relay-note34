## 💡 추가 고민과 제안

### 😶‍🌫️ AI가 대신 창의력을 발휘해주면 어떨까?

미션을 수행하다 보면 너무 피곤해서 창의적인 생각이 잘 떠오르지 않기도 합니다.  
그렇다면 굳이 개발자가 직접 창의력을 발휘하지 않아도 되는 방법은 어떨까 하는 생각이 들었습니다.

1. 아직 창의력이 익숙하지 않은 개발자들을 위해 **생성형 AI가 대신 창의력을 발휘해주는 것**
2. AI가 만든 결과물을 보며 영감을 얻고 내일도 힘내서 개발하는 선순환
3. 궁극적으로 개발자가 AI를 통해 창의력을 기를 수 있는 경험을 만든다.

이런 고민들을 바탕으로, 우리는 부담은 줄이고 자발성은 높이며, AI와 함께 즐겁고 창의적으로 성장하는 문화를 만들어가고자 합니다.

### 🎨 감성적 재해석을 위한 퀘스트 방향 제안

개발자는 다양한 이해관계자와 협업하고 소통해야 하기 때문에 표현력과 전달력은 매우 중요한 역량입니다. 그런 점에서 이 프로젝트는 AI를 활용해 지식이나 생각을 다양한 형태로 표현함으로써 창의력과 표현력을 기를 수 있는 좋은 방향이라고 생각합니다.

다만, 퀘스트 3번은 학습한 내용을 AI로 정리하고 공유하는 활동에 초점이 맞춰져 있어 프로젝트의 핵심 목표인 ‘개발자의 감성적 재해석’이라는 취지와는 다소 거리가 느껴졌습니다. 따라서 학습 내용을 보다 예술적이거나 감성적인 방식으로 풀어낼 수 있도록 보완해 본다면 프로젝트의 방향성과 더 잘 맞아떨어질 것 같습니다.

### 🖼️ 예술적 창작과 뇌 피로도

AI를 활용해 창의력을 발휘하는 시도는 분명 흥미롭고 가치 있는 활동입니다. 하지만 실제 캠퍼들의 의견을 들어보면, 예술적 표현을 직접 떠올리는 과정 자체가 오히려 피로감을 줄 수 있다는 점도 무시할 수 없습니다.

특히 퀘스트 3번처럼 학습한 내용을 정리해 자랑하는 성격의 미션은, **예술보다는 결과 중심의 과제**처럼 느껴질 수 있어 부담이 될 수 있습니다.

이런 경우에는 자유로운 창작보다는 가볍게 시각적 요소를 활용하는 방식
예를 들어 짧은 만화, 감정 이모티콘, AI 이미지 생성 도구 등을 도입하는 것이 더 효과적일 수 있다는 의견도 있었습니다.

결국 중요한 건 창의적이어야 한다는 압박이 아니라, AI를 활용해 쉬우면서도 재밌게 표현할 수 있는 방법을 찾는 것입니다.
지친 뇌에 새로운 자극이 되기 위해서는, ‘예술적인 부담’보다는 놀이 같은 느낌으로 미션을 진행하면 좋을 것 같습니다.

### 🤔퀘스트3 부담 완화 및 자발성 유도

캠퍼들이 미션 수행으로 지친 뇌를 환기하고 창의력을 기를 좋은 기회라는 데 공감했지만 퀘스트 3의 달성 기준 중 “남들에게 자랑하고 싶을 정도로 잘 했다 싶은가?”라는 문구가 부담으로 작용해 선택을 망설이는 경우가 있겠다는 생각이 들었습니다.  
다만, 퀘스트 3의 달성 기준 중 “남들에게 자랑하고 싶을 정도로 잘 했다 싶은가?”라는 문구가 부담으로 작용해 선택을 망설이는 경우가 있겠다는 의견이 나왔습니다.
따라서 “다른 사람과 공유하고 싶은 마음이 든다면 달성한 것으로 본다”와 같이 부담을 줄이고 자발성을 이끌어내는 방향으로 달성 기준을 바꾸면 좋겠다는 결론을 내렸습니다.

#### 달성조건

- 생성형 AI에게 **오늘 하루 어땠는지**를 전달하고, 내용을 **짧고 재미있는 코드**로 바꿔달라고 한다.
- 미션을 수행하면서 즐거웠던 점, 힘들었던 점, 미션이 아닌 일상 이야기 등 **주제는 뭐든 가능**하다.

#### 결과 예시

```js
// 특정 기술이 들어가지 않은 예시

  에너지 -= 30;
  커피 -= 1;

  if (커피 < 0) {
    console.log("커피 다 떨어졌어... 퇴근하고 싶다ㅠㅠ");
    break;
  }

console.log("퇴근벨 울림! 집에 가자~ 🏃‍♀️💨");
```

// 하루의 졸림 상태를 받아, 낮잠 시간이 얼마나 길어졌는지 계산하는 순수 함수

### K025 주연우

- 퀘스트: #1. 개발자는 일기도 코드로 써
- 선택한 이유: 나의 동작을 함수로 혹은 데이터로 작성해본다는 생각은 해보지 못해서, 참신하다고 생각해서 선택하였습니다.
- 목표: 아직 객체지향, 함수형 프로그래밍에 익숙하지 않은 상황입니다. 간단한 코드여도 내가 제일 잘 아는 나의 경험을 코드로 작성해보면서 여러 패러다임에 익숙해지고 싶습니다.
- **작업 과정 및 결과:**
  - #1 개발자는 일기도 코드로 써 - 그 날 잠을 못 자서, 피곤함을 나타내는 코드를 함수형 프로그래밍 패러다임으로 작성해달라고 했습니다.

```kotlin
data class DayState(val sleepHours: Int, val energy: Int, val mood: String)

val calculateEnergy: (Int) -> Int = { sleep ->
    when {
        sleep >= 7 -> 100
        sleep in 5..6 -> 70
        else -> 40
    }
}

val determineMood: (Int) -> String = { energy ->
    when {
        energy >= 80 -> "쌩쌩해요 😎"
        energy >= 50 -> "좀 피곤해요 😵"
        else -> "졸려 죽겠어요 😫"
    }
}

val buildDayState: (Int) -> DayState = { sleep ->
    val energy = calculateEnergy(sleep)
    val mood = determineMood(energy)
    DayState(sleep, energy, mood)
}

fun main() {
    val sleepLastNight = 3  // 어제 3시간밖에 못잠 😭
    val today = buildDayState(sleepLastNight)

    println("🕓 어제 수면 시간: ${today.sleepHours}시간")
    println("🔋 에너지: ${today.energy}")
    println("🧠 상태: ${today.mood}")
}
```

### J074 김채영

- 퀘스트: #1. 개발자는 일기도 코드로 써
- 선택한 이유: 일기를 코드로 표현한다는 점이 신선하고 재밌게 느껴졌습니다. 또한 AI가 써준 코드를 보면서 일기도 쓰고 학습도 될 것 같아서 선택했습니다.

- 작업 과정 및 결과

  릴레이 프로젝트로 오늘 하루를 AI를 활용해 코드로 표현했습니다.

  저는 코드 구현과 개선 작업을 해야 하는데, AI에게 **"졸리지만 그래도 하는 나"**를 코드로 표현해달라고 ChatGPT에게 요청했습니다.

  **결과**

  ```js
  const 나 = {
    에너지: 30,
    졸림: true,
    할일: ["코드 구현", "개선"],
  };

  const 실행 = (task) =>
    나.졸림 ? `😵 ${task}... 졸려도 한다...` : `😎 ${task}, 집중해서 완료!`;

  나.할일.forEach((task) => {
    console.log(실행(task));
    나.에너지 -= 10;
  });
  ```

  **실행 결과**

  ```js
  😵 코드 구현... 졸려도 한다...
  😵 개선... 졸려도 한다...
  ```

  위처럼 AI 오늘 하루의 나를 코드로 표현해주었고, 코드를 슬랙 random 채널에 공유했습니다.

  결과를 실행해보니 제 상태를 꽤 그럴듯하게 표현해줘서 웃으면서 진행했습니다.

  이처럼 간단한 프롬프트 하나로도 AI가 하루를 코드로 풀어낼 수 있다는 점이 흥미로웠고, "AI를 이런 점에서도 활용할 수 있구나" 하며 활용성을 느꼈습니다.

  그리고 간단한 프롬프트 한 줄로 AI가 단 몇 초만에 코드를 바로 작성하는 것을 보고 이렇게까지는 아니지만 "더 분발해야겠다" 라고 느꼈습니다.

  아쉬운 점이 있다면, 결과 코드가 현재 시점의 제 상태만을 단순하게 표현하고 있고 시간의 흐름이나 감정의 변화 등 하루 전체의 흐름을 담기에는 부족했다고 생각합니다.

function myBusyDayWithNaps() {

    const delay = (ms) => new Promise(res => setTimeout(res, ms));

#### 느낀점

ChatGPT에게 학습 내용을 전달하고, 그 내용을 이해하여 그림으로 바꾸는 것이 신기했습니다. 기존에 이미지를 간단하게 생성할 때는 간략하게 정보를 전달하여 이미지를 생성했었던 것 같은데, 비교적 많은 내용을 담아도 원하는 대로 이미지를 잘 생성하는 것을 보고 신기했습니다. 한글에 대한 정확도는 여전히 부족하지만요... 미션을 수행하고 난 생각이지만, ChatGPT가 아니라 조금 더 이미지 생성에 특화된 모델을 이용했으면 더 좋았을 것 같다는 아쉬움이 남네요!
